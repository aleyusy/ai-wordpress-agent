# 🤖 AI WordPress Agent

**ИИ-агент для управления WordPress через чат.** Создавайте страницы, наполняйте контентом, настраивайте дизайн, устанавливайте плагины и управляйте сайтом — просто общаясь с ИИ.

## ✨ Возможности

### 🧠 AI-интеграция
- Поддержка **OpenRouter**, **OpenAI** и любых OpenAI-совместимых API
- Выбор модели из выпадающего списка (GPT-4o, Claude, Gemini, Llama и др.)
- Авто-загрузка доступных моделей через OpenRouter API
- Настраиваемая температура и лимит токенов

### 🧠 Долговременная память
- Агент запоминает структуру сайта, предпочтения пользователя, изученные решения
- Контекст сессии сохраняется между сообщениями
- История чата хранится per-user (каждый пользователь свою)
- Автоматическое обновление памяти при изменениях на сайте

### 🎯 Роли и права доступа
- **Администратор** — полный доступ ко всем функциям
- **Редактор** — чат с агентом, создание/редактирование контента
- **Модератор** — чат с агентом (базовые действия)
- Настраиваемые права для каждой роли через интерфейс

### 🔍 Анализ сайта
- Полный аудит: темы, плагины, безопасность, скорость, SEO, контент, пользователи
- Баллы: Безопасность /100, Скорость /100, SEO /100, Общий /100
- Список проблем с уровнями серьёзности
- Рекомендации по улучшению

### 📚 Поиск плагинов и документация
- Поиск плагинов на WordPress.org
- Просмотр документации (описание, установка, FAQ, changelog)
- Проверка совместимости (версия WP, PHP, конфликты)
- Популярные плагины по категориям

### 🎯 Скиллы и воркфлоу
- Сохранение повторяющихся последовательностей действий
- Параметризация скиллов (шаблоны с плейсхолдерами)
- Условия выполнения шагов
- Экспорт/импорт скиллов между сайтами
- 3 дефолтных скилла (Landing Page, SEO, Security)

### 📝 Редактирование кода темы
- Просмотр и редактирование файлов активной темы
- Автоматические бэкапы перед сохранением
- Поддержка PHP, CSS, JS, TXT, HTML файлов
- Дерево файлов с навигацией
- Лог всех правок

### 📄 Управление контентом
- Создание и редактирование страниц и постов
- Назначение шаблонов, родительских страниц, рубрик и меток
- Загрузка изображений по URL и установка миниатюр
- Управление категориями и тегами

### 🎨 Дизайн и внешний вид
- Переключение тем оформления
- Кастомный CSS (дополнение к теме)
- Настройка theme mods (цвета, шрифты и т.д.)
- Управление меню и их расположением

### 🔌 Плагины
- Установка плагинов из WordPress.org репозитория
- Активация и деактивация
- Просмотр установленных плагинов и их статуса

### 🧩 Виджеты и сайдбары
- Просмотр зарегистрированных сайдбаров
- Добавление и удаление виджетов
- Поддержка всех типов виджетов

### ⚙️ Настройки сайта
- Изменение любых опций WordPress
- Установка домашней страницы и страницы постов
- Управление пользователями и комментариями
- Просмотр полной информации о сайте

## 🚀 Установка

### Через Git (рекомендуется)

```bash
cd /wp-content/plugins/
git clone https://github.com/aleyusy/ai-wordpress-agent.git
```

Активировать плагин в админке WordPress.

### Вручную

1. Скачайте [последний релиз](https://github.com/aleyusy/ai-wordpress-agent/releases)
2. Распакуйте архив в `/wp-content/plugins/ai-wordpress-agent/`
3. Активируйте плагин через меню **Плагины** в админке WordPress

---

## ⚡ Быстрый старт

1. Перейдите в **AI Agent → Настройки**
2. Укажите **API ключ** (получить можно на [OpenRouter.ai](https://openrouter.ai) или [platform.openai.com](https://platform.openai.com))
3. Выберите **модель** (по умолчанию `openai/gpt-4o-mini`)
4. Сохраните настройки
5. Откройте **AI Agent → Чат** и напишите:

> *«Проанализируй сайт и создай страницу "О нас"»*

## 📋 Примеры запросов

| Запрос | Что произойдёт |
|--------|----------------|
| *«Проанализируй сайт»* | Полный аудит с баллами и рекомендациями |

| *«Найди плагин для форм»* | Поиск на WordPress.org с документацией |
| *«Создай скилл для лендинга»* | Сохранён повторяемый воркфлоу |
| *«Какие права у редактора?»* | Покажет capabilities для роли |

## 🛠 Требования

- **WordPress** 5.8+
- **PHP** 7.4+
- **API ключ** OpenRouter, OpenAI или другого провайдера

---

## 🔒 Безопасность

- Действия выполняются только после аутентификации
- API ключ хранится в базе данных WordPress
-_nonce проверка для всех AJAX-запросов
- Role-based доступ: каждая роль видит только разрешённые инструменты
- Редактирование кода: sandbox только в директорию активной темы
- Автоматические бэкапы перед записью файлов
- Лог всех правок файлов
